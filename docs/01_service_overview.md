# 서비스 개요

## 서비스명

**밥심(BAB:SSIM)** - Balance And Budget Smart Meal

"밥이 보약이다"라는 말에서 착안한 이름으로, 자취생의 건강한 한 끼를 책임진다는 의미를 담았습니다.

## 문제 정의

### 자취 대학생의 영양 불균형 실태

현대 자취 대학생들은 다음과 같은 문제에 직면해 있습니다:

1. **시간 부족**
   - 학업, 과제, 아르바이트로 인한 시간 압박
   - 식사 준비에 시간을 할애하기 어려움
   - 편의점, 배달음식에 의존하는 식습관

2. **요리 경험 부족**
   - 집을 떠나 처음 독립생활을 시작하는 학생들
   - 기본적인 조리 방법조차 모르는 경우가 많음
   - "오늘 뭐 먹지?"에 대한 의사결정 피로

3. **제한된 예산**
   - 한정된 생활비로 식비 절약 필요
   - 건강한 식재료는 비싸다는 인식
   - 1인분 요리의 비효율성 (재료 남음, 폐기)

4. **영양 불균형**
   - 탄수화물 위주의 끼니 (라면, 컵밥, 김밥)
   - 야채, 단백질 섭취 부족
   - 불규칙한 식사 시간
   - 장기적으로 건강 문제로 이어질 위험

### 결과적인 악순환

```
시간 없음 → 편의점/배달 음식 → 영양 불균형 → 건강 악화 
→ 체력 저하 → 더 요리할 의욕 상실 → 반복
```

## 타겟 사용자

### 1차 타겟: 자취 대학생

**인구통계학적 특성**
- 연령: 19-26세
- 상황: 원룸/오피스텔 거주, 1인 가구
- 생활 패턴: 불규칙한 일정, 제한된 예산

**행동적 특성**
- 스마트폰 사용에 익숙함
- 빠르고 간단한 솔루션 선호
- 건강에 대한 관심은 있으나 실천이 어려움
- SNS를 통한 정보 습득

### 2차 타겟 (확장 가능성)
- 사회 초년생 자취러
- 요리 초보 신혼부부
- 혼자 사는 직장인

### 페르소나

**페르소나 1: 민준 (22세, 컴퓨터공학과 3학년)**
- 프로그래밍 과제로 밤 늦게까지 작업
- 요리 경험 전무, 라면과 냉동식품이 주식
- 한 달 식비 30만원 목표
- 최근 건강검진에서 영양 불균형 지적받음

**페르소나 2: 지은 (21세, 경영학과 2학년)**
- 오전 수업 + 오후 아르바이트로 바쁜 일정
- 간단한 요리는 할 수 있으나 메뉴 정하기 어려움
- 다이어트 관심 있으나 끼니는 거르지 않고 싶음
- 한 끼 5,000원 이하 선호

**페르소나 3: 현우 (24세, 체육교육과 4학년)**
- 운동 후 단백질 섭취 필요
- 요리는 할 줄 알지만 매일 메뉴 고민이 피곤함
- 근육 증량을 위한 영양 관리 필요
- 조리 시간 20분 이내 선호

## 핵심 가치 제안

### "3분 질문으로 오늘의 영양 만점 한 끼를 찾아드립니다"

**밥심이 제공하는 가치**

1. **의사결정 피로 해소**
   - "오늘 뭐 먹지?" 고민 종료
   - 간단한 질문에 답하면 즉시 추천
   - 선택지 제공으로 부담 없는 결정

2. **맞춤형 영양 관리**
   - 자취생에게 부족한 영양소 자동 고려
   - 개인의 상황에 맞는 식단 추천
   - 건강한 식습관 형성 지원

3. **현실적인 솔루션**
   - 예산 범위 내 추천
   - 실제 조리 가능한 수준의 레시피
   - 집에 있는 재료 활용 가능

4. **시간 절약**
   - 레시피 검색 시간 단축
   - 필요한 재료 한눈에 파악
   - 조리 시간 미리 확인 가능

### 차별화 포인트

기존 레시피 앱과의 차이:
- **검색 불필요**: 능동적 추천 방식
- **자취생 특화**: 1인분, 저예산, 간단한 조리
- **영양 우선**: 맛뿐만 아니라 영양 균형 고려
- **빠른 의사결정**: 복잡한 필터링 없이 질문 기반

## MVP 범위

### MVP에 포함되는 기능

**핵심 기능**
1. 사용자 질문 응답 시스템 (6-8개 질문)
2. 조건 기반 식단 추천 (3개 옵션 제공)
3. 추천 결과 상세 정보 제공
   - 메뉴명
   - 필요 재료 리스트
   - 간단한 조리 순서
   - 예상 조리 시간
   - 영양 정보 (칼로리, 주요 영양소)

**필수 질문 항목**
- 조리 가능 여부 (도구, 경험, 시간)
- 예산
- 기피 식재료

### MVP에서 제외되는 기능 (향후 확장)

1. 회원가입/로그인
2. 추천 히스토리 저장
3. 장보기 리스트 생성
4. 냉장고 재료 관리
5. 조리 타이머/알림
6. 사용자 리뷰 및 평가
7. 영양 섭취 통계
8. 소셜 기능 (공유, 친구 추천 등)

### 성공 기준

MVP 검증을 위한 핵심 질문:
- 사용자가 추천받은 메뉴를 실제로 만들어 먹는가?
- 영양 균형이 개선되는가?
- 반복적으로 사용하는가?
- "오늘 뭐 먹지?" 고민이 해소되는가?

## 서비스 사용 흐름 (간단 요약)

1. 앱 실행
2. 간단한 질문에 응답 (3분 이내)
3. 3개의 식단 옵션 제공받음
4. 하나 선택하여 상세 정보 확인
5. 재료 확인 후 조리 시작

**목표**: 고민 시간 최소화, 실행 가능성 최대화
