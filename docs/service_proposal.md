# 밥심(BAB:SSIM) 서비스 기획안

**자취 대학생의 영양 불균형 문제를 해결하기 위한 간단한 질문 기반 한끼 식단 추천 MVP 서비스**

---

## 1. 서비스 개요

### 서비스명
**밥심(BAB:SSIM)** - Balance And Budget Smart Meal

"밥이 보약이다"라는 말에서 착안한 이름으로, 자취생의 건강한 한 끼를 책임집니다.

### 핵심 가치
> "3분 질문으로 오늘의 영양 만점 한 끼를 찾아드립니다"

### 해결하려는 문제

**자취 대학생의 현실**
- 시간 부족 → 편의점/배달음식 의존
- 요리 경험 부족 → "오늘 뭐 먹지?" 스트레스
- 제한된 예산 → 건강한 식재료는 비싸다는 인식
- 영양 불균형 → 탄수화물 위주, 단백질/채소 부족

**악순환의 고리**
```
시간 없음 → 편의점/배달 음식 → 영양 불균형 → 건강 악화 
→ 체력 저하 → 더 요리할 의욕 상실 → 반복
```

### 타겟 사용자

**1차 타겟: 자취 대학생 (19-26세)**
- 원룸/오피스텔 거주, 1인 가구
- 불규칙한 일정, 제한된 예산
- 스마트폰 사용에 익숙, 간단한 솔루션 선호

**페르소나 예시**
- **민준**: 요리 경험 전무, 라면과 냉동식품이 주식, 월 식비 30만원
- **지은**: 간단한 요리 가능, 다이어트 관심, 한 끼 5,000원 이하 선호
- **현우**: 운동 후 단백질 필요, 매일 메뉴 고민 피곤, 조리시간 20분 이내

## 2. 서비스 기능

### MVP 핵심 기능

1. **질문 응답 시스템** (3분 이내)
   - 3개 필수 질문 + 4개 선택 질문
   
2. **식단 추천**
   - 조건에 맞는 3개 메뉴 추천
   
3. **상세 정보 제공**
   - 재료 리스트, 조리 순서, 조리 시간, 영양 정보

### 질문 항목

**필수 질문 (꼭 답변)**

1. **조리 가능 여부**
   - 조리 불가 / 10분 이내 / 20분 이내 / 30분 이상

2. **예산**
   - 3천원 이하 / 3-5천원 / 5-8천원 / 8천원 이상

3. **기피 식재료**
   - 못 먹거나 싫어하는 재료 선택/입력

**선택 질문 (생략 가능)**

4. 식사 시간대 (아침/점심/저녁/야식)
5. 집에 있는 재료
6. 영양 목표 (단백질/채소/가볍게/든든하게)
7. 요리 유형 (국물/밥/면/안주)

### 추천 결과 구성

**3개 메뉴 제공**
- 메뉴 1: 균형잡힌 기본 메뉴 (영양 균형 최우선)
- 메뉴 2: 간편한 메뉴 (조리 시간 최소화)
- 메뉴 3: 특색 있는 메뉴 (새로운 시도)

**각 메뉴 정보**
- 메뉴명 + 한 줄 소개
- 예상 조리시간
- 난이도 (★☆☆ / ★★☆ / ★★★)
- 칼로리 + 주요 영양소
- 태그 (#단백질풍부, #채소듬뿍 등)

**상세 페이지**
```
필요한 재료
- 주재료, 부재료, 양념 리스트
- 예상 비용

조리 순서
- 단계별 설명 (5단계 이내)
- 총 소요시간

영양 정보
- 칼로리, 탄수화물, 단백질, 지방
- 풍부한 영양소

조리 팁 & 주의사항
- 초보자를 위한 팁
- 보관 방법
```

## 3. 추천 기준

### 필수 조건 (반드시 충족)

1. **조리 능력 범위 내**
   - 사용자가 선택한 조리 시간 초과 메뉴 제외
   
2. **예산 범위 내**
   - 선택한 예산 초과 메뉴 완전 제외
   
3. **기피 식재료 완전 제외**
   - 해당 재료 포함 메뉴는 추천 안 함

### 우선순위 (점수 부여)

- 집에 있는 재료 활용: +20점
- 영양 균형도: +25점
- 영양 목표 부합: +15점
- 요리 유형 일치: +10점
- 식사 시간대 적합: +10점
- 다양성 (중복 방지): +20점

### 영양 균형 기준

**이상적인 한 끼**
- 탄수화물 50-60%
- 단백질 20-30% (15-25g)
- 지방 20-30%
- 채소 200g 이상

**칼로리 가이드**
- 아침: 400-500kcal
- 점심/저녁: 600-800kcal
- 야식: 300-400kcal

## 4. 사용자 시나리오 예시

### 시나리오: 요리 초보 민준 (22세)

**상황**: 저녁 7시, 과제 끝. 배고프지만 요리 경험 전무

**질문 응답**
- 조리 가능: 10분 이내 간단 조리
- 예산: 3-5천원
- 기피 재료: 없음
- 시간대: 저녁
- 집에 있는 재료: 계란, 김치, 밥
- 영양 목표: 단백질 보충

**추천 결과**
1. 김치계란볶음밥 (10분, ★☆☆, 520kcal, 단백질 18g)
2. 참치김치볶음밥 (12분, ★☆☆, 580kcal, 단백질 22g)
3. 계란덮밥 (8분, ★☆☆, 450kcal, 단백질 15g)

**결과**: 김치계란볶음밥 선택, 실제 15분 소요, 성공! 만족도 ⭐⭐⭐⭐☆

## 5. 기대 효과

### 문제 해결

| 문제 | 해결 방안 | 기대 효과 |
|------|----------|----------|
| 의사결정 피로 | 3분 질문으로 즉시 추천 | 메뉴 고민 20분→3분 |
| 영양 불균형 | 단백질/채소 포함 메뉴 우선 | 채소 섭취 주 2회→5회 |
| 요리 장벽 | 능력에 맞는 메뉴만 추천 | 요리 시도 주 0회→3회 |
| 비용 부담 | 예산 범위 내 추천 | 월 식비 30-40% 절감 |

### 사용자 경험 개선

**Before (서비스 사용 전)**
- 저녁 6-7시: "뭐 먹지?" 고민 + 배달앱 스크롤
- 저녁 7시: 배달 주문 (1만원+)
- 저녁 8시: 과식, 속 더부룩, 죄책감

**After (서비스 사용 후)**
- 저녁 6시: 앱에서 3분 질문 응답
- 저녁 6시 10분: 조리 시작
- 저녁 6시 30분: 완성, 만족스러운 식사
- 시간 절약 + 돈 절약 + 건강 개선 + 뿌듯함

## 6. MVP 성공 지표

### 3개월 내 목표

**사용성 검증 (1주)**
- 질문 완료율 70% 이상
- 평균 소요시간 5분 이내

**만족도 검증 (2주)**
- 재사용률 50% 이상
- 만족도 평균 4.0/5.0 이상

**실행 검증 (1개월)**
- 조리 시도율 60% 이상
- 주간 사용 평균 3회 이상

**가치 검증 (3개월)**
- 활성 사용자(MAU) 500명 이상
- 3개월 유지율 40% 이상
- NPS 40 이상

### 성공 정의

**핵심 질문**: "이 서비스가 자취생의 영양 불균형 문제를 실제로 해결하는가?"

**검증 항목**
- ✅ 추천받은 메뉴를 실제로 만들어 먹는가?
- ✅ 영양 섭취가 개선되는가?
- ✅ 지속적으로 사용하는가?
- ✅ 주변에 추천하는가?

→ **4가지 모두 YES = MVP 성공**

## 9. 차별화 포인트

### 기존 레시피 앱과의 차이

| 구분 | 기존 레시피 앱 | 밥심 |
|------|-------------|------|
| 방식 | 검색 + 필터링 | 질문 기반 능동 추천 |
| 타겟 | 일반 사용자 | 자취 대학생 특화 |
| 우선순위 | 맛, 인기도 | 영양 균형, 실현 가능성 |
| 의사결정 | 복잡한 선택 | 3분 질문 → 즉시 추천 |
| 레시피 | 다양하지만 복잡 | 1인분, 간단, 저예산 |

### 핵심 강점
1. **빠른 의사결정**: 3분이면 끝
2. **실현 가능성**: 실제로 만들 수 있는 메뉴만
3. **영양 우선**: 자취생 부족 영양소 자동 고려
4. **예산 준수**: 선택한 금액 절대 초과 안 함

## 마무리

### 서비스의 궁극적 목표

> **"모든 자취생이 건강한 한 끼를 간편하게 즐기는 세상"**

단순히 메뉴를 추천하는 것이 아니라:
- 사용자의 **삶의 질을 개선**하고
- **건강한 식습관**을 형성하며
- **요리에 대한 자신감**을 심어주고
- **경제적 부담**을 줄이는 것

이것이 밥심의 진정한 가치입니다.
