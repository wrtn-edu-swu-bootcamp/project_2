# 기대 효과 및 성공 지표

## 서비스를 통해 해결하려는 문제

### 1. 의사결정 피로 (Decision Fatigue)

**문제**:
- 매일 "오늘 뭐 먹지?"를 고민하는 스트레스
- 메뉴 선택에 소요되는 시간과 에너지
- 결국 항상 같은 음식으로 귀결 (라면, 배달음식)

**해결 방안**:
- 3분 내 간단한 질문으로 즉시 추천
- AI가 대신 생각해주는 방식
- 3가지 옵션 제공으로 부담 없는 선택

**기대 효과**:
- 메뉴 고민 시간 20분 → 3분으로 단축
- 정신적 스트레스 감소
- 에너지를 더 중요한 일(학업, 업무)에 집중

---

### 2. 영양 불균형

**문제**:
- 탄수화물 위주 식단 (라면, 빵, 밥)
- 단백질 섭취 부족
- 채소 섭취 거의 없음
- 불규칙한 식사 → 건강 악화 악순환

**해결 방안**:
- 영양 균형을 고려한 메뉴 추천
- 단백질, 채소가 포함된 메뉴 우선 제공
- 부족한 영양소를 자연스럽게 섭취

**기대 효과**:
- 하루 단백질 섭취량 30% 증가
- 채소 섭취 빈도 주 2회 → 주 5회 증가
- 3개월 후 건강 지표 개선 (체력, 면역력)
- 장기적으로 질병 예방

---

### 3. 요리 장벽

**문제**:
- 요리 경험 부족으로 시도 자체를 포기
- "요리는 어렵다"는 선입견
- 레시피를 봐도 이해하기 어려움
- 실패에 대한 두려움

**해결 방안**:
- 조리 능력에 맞는 메뉴만 추천
- 초보자도 따라할 수 있는 친절한 레시피
- 성공 확률 높은 메뉴 위주
- 단계별 레벨업 유도

**기대 효과**:
- 요리 시도율 증가 (주 0회 → 주 3-4회)
- "나도 요리할 수 있다"는 자신감 획득
- 요리 실력 향상 → 난이도 높은 메뉴 도전
- 요리를 취미로 즐기게 됨

---

### 4. 비용 부담

**문제**:
- 배달음식 비용 부담 (1만원 이상)
- 편의점 도시락도 반복하면 비쌈 (5-7천원)
- 한정된 생활비로 식비 절약 필요
- 건강한 음식은 비싸다는 인식

**해결 방안**:
- 예산 범위 내 메뉴만 추천
- 저렴한 재료로 영양가 높은 메뉴 제안
- 집에 있는 재료 활용
- 1인분 레시피로 음식물 쓰레기 최소화

**기대 효과**:
- 월 식비 30-40% 절감
- 배달음식 주 5회 → 주 1-2회 감소
- 한 끼 평균 비용 10,000원 → 5,000원 절감
- 절약한 돈으로 다른 곳에 투자 가능

---

### 5. 음식물 쓰레기

**문제**:
- 1인 가구 특성상 재료 남음
- 남은 재료 활용 방법 몰라서 버림
- 냉장고에 방치된 재료 부패

**해결 방안**:
- 집에 있는 재료 입력 → 해당 재료 활용 메뉴 추천
- 1인분 레시피로 딱 필요한 만큼만 구매
- 재료 활용 팁 제공

**기대 효과**:
- 음식물 쓰레기 50% 감소
- 재료 활용도 증가
- 환경 보호에 기여
- 불필요한 재료 구매 감소 → 비용 절약

---

### 6. 불규칙한 식습관

**문제**:
- 끼니 거르기 일쑤
- 야식, 배달음식으로 대충 때움
- "귀찮아서" 안 먹거나 과식

**해결 방안**:
- 간편한 메뉴 추천으로 조리 장벽 낮춤
- 10분 내 완성 가능한 메뉴도 제공
- 식사 시간대에 맞는 메뉴 추천

**기대 효과**:
- 규칙적인 식사 습관 형성
- 끼니 거르는 빈도 감소
- 전반적인 건강 개선
- 체력 및 집중력 향상

## 사용자 경험 개선 포인트

### 1. 심리적 개선

**Before** (서비스 사용 전):
- 😰 메뉴 고민 스트레스
- 😔 요리 자신감 부족
- 😟 건강 악화 걱정
- 😤 배달비 아까움

**After** (서비스 사용 후):
- 😊 메뉴 고민 해결
- 💪 요리 자신감 향상
- 🥗 건강해지는 느낌
- 💰 절약하는 뿌듯함
- 🎉 새로운 메뉴 도전 재미

### 2. 생활 패턴 개선

**Before**:
```
저녁 6시: 집 도착
저녁 6시-7시: "뭐 먹지?" 고민 + 배달앱 스크롤
저녁 7시: 배달 주문
저녁 7시 30분: 배달 도착, 식사
저녁 8시: 과식으로 속 더부룩
저녁 9시: 죄책감 + 건강 걱정
```

**After**:
```
저녁 6시: 집 도착
저녁 6시-6시 3분: 앱에서 메뉴 추천받기
저녁 6시 5분: 재료 확인 (또는 편의점 방문)
저녁 6시 10분: 조리 시작
저녁 6시 30분: 완성, 식사
저녁 7시: 만족스러운 식사 완료
저녁 7시 이후: 여유 시간 + 뿌듯함
```

**개선 효과**:
- 시간 절약 (30분 → 3분)
- 돈 절약 (만원 → 5천원)
- 건강 개선 (영양 균형)
- 심리 만족 (성취감)

## MVP 검증을 위한 성공 지표

### 1단계: 사용성 검증 (출시 후 1주)

**목표**: 사용자가 서비스를 쉽게 사용할 수 있는가?

| 지표 | 측정 방법 | 성공 기준 |
|------|----------|----------|
| **질문 완료율** | 마지막 질문까지 답변한 비율 | 70% 이상 |
| **평균 소요 시간** | 질문 시작 ~ 추천 결과까지 | 5분 이내 |
| **추천 결과 도달률** | 추천 화면을 본 사용자 비율 | 80% 이상 |
| **메뉴 상세 조회율** | 추천 메뉴 중 1개 이상 클릭 | 90% 이상 |
| **이탈률** | 중간에 앱을 닫은 비율 | 30% 이하 |

**검증 방법**:
- 앱 내 이벤트 추적
- 화면 전환 로그 분석

---

### 2단계: 만족도 검증 (출시 후 2주)

**목표**: 사용자가 추천에 만족하는가?

| 지표 | 측정 방법 | 성공 기준 |
|------|----------|----------|
| **재사용률** | 2회 이상 사용한 사용자 비율 | 50% 이상 |
| **만족도 점수** | 추천 후 "만족하셨나요?" 설문 | 평균 4점/5점 |
| **다시 추천 요청률** | "다른 메뉴 보기" 클릭 비율 | 30% 이하 * |
| **앱 평점** | 앱스토어 리뷰 평점 | 4.0/5.0 이상 |

\* 낮을수록 첫 추천에 만족했다는 의미

**검증 방법**:
- 사용자 행동 패턴 분석
- 간단한 만족도 팝업 (추천 후 1회만)
- 앱스토어 리뷰 모니터링

---

### 3단계: 실행 검증 (출시 후 1개월)

**목표**: 사용자가 실제로 요리를 만들어 먹는가?

| 지표 | 측정 방법 | 성공 기준 |
|------|----------|----------|
| **조리 시도율** | "만들어봤어요" 피드백 비율 | 60% 이상 |
| **조리 성공률** | "성공했어요" vs "실패했어요" | 80% 이상 |
| **주간 사용 빈도** | 일주일에 사용하는 횟수 | 평균 3회 이상 |
| **추천 수락률** | 추천받은 메뉴를 실제 조리 | 60% 이상 |

**검증 방법**:
- 메뉴 상세 화면 하단에 "만들어봤어요" 버튼
- 간단한 후기 (성공/실패, 별점)
- 주간 리포트 발송 후 응답률

---

### 4단계: 가치 검증 (출시 후 2-3개월)

**목표**: 사용자의 삶이 실제로 개선되는가?

| 지표 | 측정 방법 | 성공 기준 |
|------|----------|----------|
| **식비 절감** | 사용자 설문 | 월 평균 10만원 절감 |
| **요리 빈도 증가** | "이전보다 요리를 자주 하나요?" | 70% "그렇다" |
| **영양 개선 체감** | "건강해졌다고 느끼나요?" | 60% "그렇다" |
| **추천 의향** | "친구에게 추천하겠는가?" (NPS) | NPS 40 이상 |
| **유지율** | 3개월 후에도 사용하는 비율 | 40% 이상 |

**검증 방법**:
- 월간 설문조사 (간단한 3-5개 질문)
- NPS(Net Promoter Score) 조사
- 코호트 분석

- **NPS**: 40 이상
- **재사용률**: 50% 이상
- **3개월 유지율**: 40% 이상

## 장기 비전 (1년 후)

### 사회적 임팩트

1. **자취생 건강 개선**
   - 영양 불균형 문제 해결
   - 건강한 식습관 형성
   - 만성질환 예방

2. **식비 절감**
   - 배달음식 의존도 감소
   - 경제적 부담 완화
   - 합리적 소비 문화

3. **환경 보호**
   - 음식물 쓰레기 감소
   - 일회용 용기 사용 감소
   - 지속 가능한 식문화

4. **요리 문화 확산**
   - 요리에 대한 긍정적 인식
   - 청년층 요리 실력 향상
   - 가정간편식(HMR) 의존도 감소

### 서비스 확장

**Phase 1** (MVP): 기본 추천 기능
**Phase 2** (3개월): 히스토리, 장보기 리스트
**Phase 3** (6개월): 냉장고 관리, 영양 대시보드
**Phase 4** (1년): 커뮤니티, AI 학습 고도화

## 실패 시나리오 및 대응

### 위험 요소

1. **추천 정확도 부족**
   - 위험: 사용자가 추천을 신뢰하지 않음
   - 대응: 메뉴 DB 확장, 필터링 로직 개선

2. **조리 실패 경험**
   - 위험: 레시피 따라했는데 실패 → 재사용 안 함
   - 대응: 레시피 단순화, 동영상 튜토리얼 추가

3. **메뉴 다양성 부족**
   - 위험: 같은 메뉴만 반복 추천 → 지루함
   - 대응: 메뉴 DB 확장 (최소 50개 → 100개)

4. **예산/재료 가격 변동**
   - 위험: 실제 가격이 예상보다 높음
   - 대응: 정기적인 가격 업데이트, 사용자 피드백 반영

5. **사용자 참여 저조**
   - 위험: 설치 후 1회만 사용하고 이탈
   - 대응: 푸시 알림, 리마인더, 사용 유도 이벤트

## 성공 정의

### MVP 성공 기준 (3개월 내 달성)

다음 중 **3가지 이상** 충족 시 성공으로 판단:

1. ✅ **활성 사용자 500명** 이상 확보
2. ✅ **재사용률 50%** 이상
3. ✅ **만족도 4.0/5.0** 이상
4. ✅ **조리 시도율 60%** 이상
5. ✅ **NPS 40** 이상
6. ✅ **3개월 유지율 40%** 이상

### 궁극적 성공 지표

**핵심 질문**: 
> "이 서비스가 자취생의 영양 불균형 문제를 실제로 해결하는가?"

**검증 방법**:
- 사용자가 추천받은 메뉴를 실제로 만들어 먹는가? (YES)
- 사용자의 영양 섭취가 개선되는가? (YES)
- 사용자가 지속적으로 사용하는가? (YES)
- 사용자가 주변에 추천하는가? (YES)

→ **4가지 모두 YES라면 MVP 성공**

### MVP 성공 시

1. **기능 확장**
   - 회원가입/로그인 추가
   - 히스토리 저장
   - 장보기 리스트

2. **메뉴 DB 확장**
   - 50개 → 100개 → 200개
   - 다양한 요리 유형
   - 계절별 메뉴

3. **마케팅 강화**
   - 대학교 커뮤니티 홍보
   - 인플루언서 협업
   - 입소문 마케팅

4. **수익화 모델 검토**
   - 프리미엄 메뉴
   - 재료 배송 제휴
   - 구독 서비스

### MVP 실패 시

1. **원인 분석**
   - 어느 단계에서 이탈하는가?
   - 왜 재사용하지 않는가?
   - 추천이 부정확한가?

2. **피벗 검토**
   - 타겟 변경 (대학생 → 직장인)
   - 서비스 방향 변경
   - 기능 대폭 수정

3. **최소 개선 후 재도전**
   - 핵심 문제만 해결
   - 재검증

## 마무리

이 서비스의 궁극적인 목표는:

> **"모든 자취생이 건강한 한 끼를 간편하게 즐기는 세상"**

단순히 메뉴를 추천하는 것이 아니라, 
- 사용자의 **삶의 질을 개선**하고
- **건강한 식습관**을 형성하며
- **요리에 대한 자신감**을 심어주고
- **경제적 부담**을 줄이는 것

이것이 우리 서비스의 진정한 가치입니다.

MVP를 통해 이 가치를 검증하고, 실제로 사용자의 삶을 변화시킬 수 있는지 확인합니다.
