# 사용자 질문 Flow

## 질문 설계 원칙

1. **간결성**: 3분 이내 완료 가능한 질문 수
2. **명확성**: 전문 용어 배제, 쉬운 언어 사용
3. **유연성**: 모든 질문에 "잘 모르겠어요" 옵션 제공
4. **순차성**: 답변에 따라 다음 질문 조정 가능

## 질문 구성

### 필수 질문 (3개)

사용자가 반드시 답해야 하는 핵심 질문입니다.

---

#### Q1. 조리 가능 여부

**질문 배경**: 사용자의 요리 환경과 역량을 파악하여 실현 가능한 메뉴만 추천

**질문 문구**: "오늘 요리 준비, 어느 정도 가능하신가요?"

**선택지**:
1. **조리 불가** 
   - "요리 못해요 / 시간 없어요"
   - → 조리 불필요 메뉴 추천 (편의점 재료 조합, 간단 조리)
   
2. **10분 이내 간단 조리**
   - "계란 프라이, 라면 끓이기 정도는 가능해요"
   - 필요 도구: 가스레인지/인덕션, 냄비/프라이팬 1개
   - → 1-2가지 재료, 단순 조리법 추천
   
3. **20분 이내 기본 조리**
   - "볶고 끓이는 정도는 할 수 있어요"
   - 필요 도구: 기본 조리도구 보유
   - → 다양한 메뉴 추천 가능
   
4. **30분 이상 본격 조리**
   - "요리 좋아해요! 시간 여유 있어요"
   - → 모든 메뉴 추천 가능

**후속 정보 수집** (선택지에 따라):
- 조리 불가 선택 시: 근처 편의점 접근 가능 여부
- 조리 가능 선택 시: 보유 조리 도구 간단 체크 (선택사항)

---

#### Q2. 예산

**질문 배경**: 실현 가능한 가격대의 메뉴만 추천하여 실행률 높임

**질문 문구**: "한 끼에 얼마까지 쓸 수 있으세요?"

**선택지**:
1. **3,000원 이하**
   - 초절약 모드
   - 재료: 계란, 라면, 김치, 밥 등 기본 식재료
   
2. **3,000원 - 5,000원**
   - 자취생 평균 예산
   - 재료: 위 + 채소 1-2가지, 저렴한 단백질 (두부, 참치캔 등)
   
3. **5,000원 - 8,000원**
   - 여유 있는 한 끼
   - 재료: 신선한 채소, 육류/생선 소량 포함 가능
   
4. **8,000원 이상**
   - 프리미엄 한 끼
   - 재료: 품질 좋은 재료 자유롭게 선택

**추가 설명**:
- "재료비만 계산해요. 이미 집에 있는 기본 양념(소금, 설탕, 간장 등)은 제외!"
- 표시 가격은 편의점/마트 평균 가격 기준

---

#### Q3. 기피 식재료

**질문 배경**: 알레르기, 종교적/개인적 이유로 못 먹는 재료 제외

**질문 문구**: "못 먹거나 싫어하는 재료가 있나요?"

**입력 방식**: 
- **자유 입력** 또는 **자주 기피되는 재료 체크리스트**

**자주 기피되는 재료 체크리스트**:
- [ ] 해산물 (새우, 오징어, 조개 등)
- [ ] 갑각류 (게, 가재 등)
- [ ] 유제품 (우유, 치즈, 버터 등)
- [ ] 달걀
- [ ] 땅콩/견과류
- [ ] 돼지고기
- [ ] 소고기
- [ ] 닭고기
- [ ] 매운 음식
- [ ] 향이 강한 채소 (파, 마늘, 셀러리 등)
- [ ] 버섯
- [ ] 나물류

**추가 옵션**:
- "직접 입력하기" 버튼 (위 리스트에 없는 재료)
- "없어요" 버튼 (기피 재료 없음)

**처리 방식**:
- 체크된 모든 재료를 포함한 메뉴는 추천에서 완전히 제외
- 대체 재료 제안 (예: 새우 싫으면 → 닭가슴살로 대체)

---

### 추가 질문 (선택적, 3-5개)

추천 정확도를 높이기 위한 부가 질문입니다. 답변 생략 가능.

---

#### Q4. 식사 시간대

**질문 문구**: "언제 드실 예정이에요?"

**선택지**:
1. **아침** (07:00-10:00)
   - 간단하고 가벼운 메뉴
   - 준비 시간 짧은 메뉴 우선
   
2. **점심** (11:00-14:00)
   - 든든한 한 끼
   - 탄수화물 + 단백질 균형
   
3. **저녁** (17:00-20:00)
   - 가장 다양한 메뉴
   - 영양 균형 중시
   
4. **야식** (20:00 이후)
   - 가볍고 부담 없는 메뉴
   - 소화 잘 되는 음식 추천

---

#### Q5. 집에 있는 재료

**질문 문구**: "집에 이미 있는 재료가 있나요?"

**입력 방식**: 
- 자유 입력 (쉼표로 구분)
- 예시: "계란, 양파, 김치, 햄"

**활용**:
- 해당 재료를 활용한 메뉴 우선 추천
- 추가 구매 재료 최소화
- 음식물 쓰레기 감소

**선택사항**: 
- "잘 모르겠어요" 선택 시 이 정보 없이 추천

---

#### Q6. 영양 목표

**질문 문구**: "오늘은 어떤 영양소를 채우고 싶으세요?"

**선택지**:
1. **단백질 보충**
   - "고기/생선/계란 먹고 싶어요"
   - 단백질 20g 이상 메뉴
   
2. **채소 섭취**
   - "요즘 야채를 못 먹었어요"
   - 채소 2가지 이상 포함 메뉴
   
3. **가볍게**
   - "다이어트 중이에요"
   - 500kcal 이하 저칼로리 메뉴
   
4. **든든하게**
   - "체력 보충이 필요해요"
   - 700kcal 이상 고칼로리 메뉴
   
5. **상관없어요**
   - 균형 잡힌 일반 메뉴

---

#### Q7. 특별한 요구사항

**질문 문구**: "그 외 원하는 게 있다면?"

**선택지**:
1. **국물 요리**
   - 국, 찌개, 탕 종류
   
2. **밥 요리**
   - 덮밥, 볶음밥, 비빔밥
   
3. **면 요리**
   - 파스타, 라면, 우동, 국수
   
4. **간단한 안주**
   - 술과 곁들일 수 있는 메뉴
   
5. **상관없어요**

---

## 질문 플로우 다이어그램

```
[시작]
  ↓
[Q1: 조리 가능 여부] ← 필수
  ↓
[Q2: 예산] ← 필수
  ↓
[Q3: 기피 식재료] ← 필수
  ↓
[Q4: 식사 시간대] ← 선택
  ↓
[Q5: 집에 있는 재료] ← 선택
  ↓
[Q6: 영양 목표] ← 선택
  ↓
[Q7: 특별한 요구사항] ← 선택
  ↓
[식단 추천 결과]
```

## 예상 소요 시간

- **최소 시간** (필수 질문만): 약 1-2분
- **권장 시간** (모든 질문 답변): 약 3-4분
- **목표**: 5분을 넘지 않도록 설계

## 사용자 경험 고려사항

### 1. 진행 상황 표시
- "3개 중 1번째 질문이에요" 형태로 표시
- 프로그레스 바 제공

### 2. 되돌아가기 기능
- 이전 질문으로 돌아가 답변 수정 가능
- "이전" 버튼 항상 표시

### 3. 건너뛰기 옵션
- 선택적 질문은 "건너뛰기" 버튼 제공
- 필수 질문은 건너뛰기 불가

### 4. 도움말
- 각 질문에 "?" 아이콘 → 설명 팝업
- 선택지 이해를 돕는 예시 제공

### 5. 기본값 설정
- 처음 사용자: 모든 질문 표시
- 재사용자: 이전 답변 기억하여 수정만 가능하게 (선택사항)

## 질문 최적화 전략

MVP 이후 데이터 수집을 통한 개선:

1. **응답률 분석**
   - 건너뛰기가 많은 질문 재검토
   - 답변이 추천 정확도에 미치는 영향 분석

2. **질문 순서 최적화**
   - 이탈률이 높은 구간 파악
   - 중요도에 따라 순서 재배치

3. **선택지 개선**
   - "기타" 응답 내용 분석하여 선택지 추가
   - 헷갈리는 선택지 문구 수정

4. **조건부 질문 적용**
   - 이전 답변에 따라 불필요한 질문 생략
   - 예: "조리 불가" 선택 시 조리 시간 질문 생략
