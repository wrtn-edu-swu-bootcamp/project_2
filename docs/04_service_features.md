# 서비스 기능 명세

## MVP 핵심 기능

### 1. 질문 응답 시스템

**기능 설명**: 사용자에게 몇 가지 질문을 하여 상황을 파악하는 기능

**세부 기능**:

1. **질문 화면 표시**
   - 한 번에 한 개의 질문만 표시 (단계별 진행)
   - 명확한 선택지 제공
   - 진행 상황 표시 (예: "7개 중 3번째")

2. **답변 입력 방식**
   - 객관식 선택 (버튼/라디오 버튼)
   - 체크박스 (기피 식재료 등)
   - 자유 입력 (집에 있는 재료 등)

3. **네비게이션**
   - "다음" 버튼: 다음 질문으로 이동
   - "이전" 버튼: 이전 질문으로 돌아가기
   - "건너뛰기" 버튼: 선택적 질문 생략

4. **도움말**
   - 각 질문에 "?" 아이콘
   - 탭하면 설명 팝업 표시
   - 예시 제공

5. **입력 검증**
   - 필수 질문 미답변 시 경고
   - 유효하지 않은 입력 방지

**사용자 경험 포인트**:
- 질문이 부담스럽지 않도록 짧고 명확하게
- 프로그레스 바로 끝이 보이도록
- 언제든 수정 가능하다는 안심감 제공

### 2. 식단 추천

**기능 설명**: 사용자의 답변을 분석하여 최적의 식단 3개를 추천

**세부 기능**:

1. **추천 결과 생성**
   - 사용자 답변 기반 필터링
   - 영양 균형 고려한 메뉴 선정
   - 다양성을 고려한 3개 메뉴 조합

2. **추천 결과 표시**
   - 카드 형태로 3개 메뉴 나란히 표시
   - 각 카드에 핵심 정보 요약 표시
   - 스크롤 또는 스와이프로 탐색

3. **메뉴 카드 정보 구성**
   - **메뉴 이미지** (음식 사진)
   - **메뉴명**
   - **한 줄 소개** (추천 이유)
   - **조리시간** (예: 15분)
   - **난이도** (★☆☆ 형태)
   - **칼로리** (예: 650kcal)
   - **태그** (#단백질풍부, #채소듬뿍 등)

4. **추천 이유 표시**
   - 왜 이 메뉴를 추천했는지 간단히 설명
   - 예: "집에 있는 김치를 활용해요"
   - 예: "단백질 25g으로 든든해요"

**사용자 경험 포인트**:
- 3개 메뉴를 한눈에 비교할 수 있도록
- 시각적으로 매력적인 음식 이미지
- 선택에 도움되는 충분한 정보 제공

### 3. 메뉴 상세 정보

**기능 설명**: 선택한 메뉴의 자세한 정보를 제공

**세부 기능**:

1. **재료 정보**
   ```
   [필요한 재료]
   
   주재료:
   - 돼지고기 (앞다리살) 200g
   - 양파 1/2개
   - 대파 1/2대
   
   양념/조미료:
   - 고추장 1스푼
   - 간장 1스푼
   - 설탕 1/2스푼
   - 다진 마늘 1스푼
   - 참기름 약간
   
   [예상 비용]
   총 약 6,500원
   (마트 평균 가격 기준)
   ```

2. **조리 방법**
   ```
   [조리 순서]
   
   1단계: 재료 준비 (5분)
   - 돼지고기를 한입 크기로 자르기
   - 양파는 채썰기, 대파는 어슷썰기
   
   2단계: 양념 만들기 (2분)
   - 볼에 고추장, 간장, 설탕, 다진 마늘 섞기
   
   3단계: 볶기 (8분)
   - 팬에 기름을 두르고 돼지고기 볶기
   - 고기가 익으면 양념 넣고 볶기
   - 양파, 대파 넣고 함께 볶기
   
   4단계: 완성
   - 참기름 살짝 두르기
   - 밥과 함께 담아내기
   
   [총 소요시간: 15분]
   ```

3. **영양 정보**
   ```
   [영양 정보]
   
   칼로리: 680kcal
   
   주요 영양소:
   - 탄수화물: 45g (밥 포함)
   - 단백질: 28g ⭐ (1일 권장량의 50%)
   - 지방: 18g
   - 식이섬유: 3g
   
   풍부한 영양소:
   - 비타민 B1 (돼지고기)
   - 비타민 C (양파)
   - 단백질 (돼지고기)
   ```

4. **팁 & 주의사항**
   ```
   [조리 팁]
   - 고기를 센 불에서 빠르게 볶으면 부드러워요
   - 양파를 먼저 볶으면 단맛이 더 나요
   - 매운 게 싫으면 고추장을 반으로 줄이세요
   
   [보관 방법]
   - 냉장 보관: 2일
   - 냉동 보관: 1주일
   
   [이럴 때 추천]
   - 고기가 당길 때
   - 든든한 한 끼가 필요할 때
   - 밥반찬으로 좋아요
   ```

5. **재료 구매 정보**
   - 주요 재료 구입처 안내
   - 대체 가능한 재료 제안
   - 예: "앞다리살 → 삼겹살 또는 목살로 대체 가능"

**사용자 경험 포인트**:
- 정보가 많지만 잘 구조화되어 읽기 쉽게
- 실제 요리할 때 참고하기 좋은 형식
- 초보자도 따라할 수 있는 친절한 설명

### 4. 다시 추천받기

**기능 설명**: 추천받은 메뉴가 마음에 들지 않으면 다시 추천받기

**세부 기능**:

1. **전체 다시 추천**
   - "다른 메뉴 보기" 버튼
   - 같은 조건으로 다른 3개 메뉴 추천
   - 이전에 본 메뉴는 제외

2. **조건 수정**
   - "질문 다시 하기" 버튼
   - 질문 화면으로 돌아가기
   - 이전 답변이 기본값으로 채워져 있음
   - 원하는 부분만 수정 가능

3. **무한 추천**
   - 만족할 때까지 계속 추천 가능
   - 단, 메뉴 DB에서 추천 가능한 메뉴가 고갈되면 안내

**사용자 경험 포인트**:
- 선택의 자유 제공
- 부담 없이 여러 옵션 탐색 가능
- 완벽한 메뉴를 찾을 때까지 지원

## MVP 제외 기능 (향후 확장)

### Phase 2: 개인화 기능

1. **회원가입 / 로그인**
   - 소셜 로그인 (카카오, 구글 등)
   - 간편 가입

2. **추천 히스토리**
   - 과거에 추천받은 메뉴 기록
   - "이전에 만들어봤어요" 표시
   - 좋아하는 메뉴 저장

3. **내 정보 저장**
   - 자주 사용하는 질문 답변 저장
   - 알레르기 정보 등록
   - 식습관 프로필 생성

### Phase 3: 편의 기능

4. **장보기 리스트**
   - 선택한 메뉴의 재료 리스트 생성
   - 집에 있는 재료 체크
   - 구매할 재료만 정리
   - 리스트 공유 기능

5. **냉장고 재료 관리**
   - 현재 보유 재료 등록
   - 유통기한 관리
   - 소진 필요한 재료 우선 추천

6. **조리 타이머**
   - 단계별 타이머 설정
   - 알림 기능
   - 다음 단계 자동 표시

### Phase 4: 소셜 기능

7. **메뉴 리뷰 및 평가**
   - 별점 및 후기 작성
   - 조리 난이도 피드백
   - 사진 업로드

8. **공유 기능**
   - 추천받은 메뉴 공유
   - 친구에게 메뉴 추천
   - SNS 공유

9. **커뮤니티**
   - 레시피 개선 제안
   - 조리 팁 공유
   - Q&A

### Phase 5: 고도화 기능

10. **영양 관리 대시보드**
    - 주간/월간 섭취 영양소 통계
    - 부족한 영양소 알림
    - 식습관 분석 리포트

11. **개인 맞춤 추천 개선**
    - AI 학습을 통한 취향 파악
    - 계절별 제철 재료 추천
    - 날씨에 따른 메뉴 추천

12. **구독 서비스**
    - 주간 식단표 자동 생성
    - 맞춤 레시피북 제공
    - 프리미엄 메뉴 추천

```

    ↓

- 서비스 소개 (처음만)
- "시작하기" 버튼
    ↓
[질문 1: 조리 가능 여부]
    ↓
[질문 2: 예산]
    ↓
[질문 3: 기피 식재료]
    ↓
[질문 4: 식사 시간대] (선택)
    ↓
[질문 5: 집에 있는 재료] (선택)
    ↓
[질문 6: 영양 목표] (선택)
    ↓
[질문 7: 특별 요구사항] (선택)
    ↓

- 팁
    ↓
```

## 각 화면별 주요 요소

### 1. 웰컴 화면

**표시 내용**:
- 서비스 로고
- 캐치프레이즈: "3분 질문으로 오늘의 영양 만점 한 끼"
- 주요 기능 소개 (아이콘 + 한 줄)
  - 빠른 추천
  - 영양 균형
  - 현실적인 레시피
- "시작하기" 버튼 (CTA)

**처음 사용자에게만**:
- 간단한 튜토리얼 슬라이드 (3장)
- 건너뛰기 가능

### 2. 질문 화면 (공통)

**상단**:
- 프로그레스 바 (현재 진행도)
- 질문 번호/총 질문 수
- 뒤로 가기 버튼

**중앙**:
- 질문 텍스트 (큰 글씨)
- 부가 설명 (작은 글씨, 선택적)
- 선택지 버튼/입력창

**하단**:
- "다음" 버튼 (답변 선택 시 활성화)
- "건너뛰기" 버튼 (선택적 질문만)

### 3. 추천 결과 화면

**상단**:
- "이런 메뉴 어때요?" 제목
- 추천 기준 요약 (예: "예산 5천원 / 20분 조리")

**메인 영역**:
- 3개 메뉴 카드 (수평 스크롤 또는 그리드)
- 각 카드: 이미지, 메뉴명, 핵심 정보

**하단**:
- "다른 메뉴 보기" 버튼
- "질문 다시 하기" 링크

### 4. 메뉴 상세 화면

**상단**:
- 뒤로 가기 버튼
- 공유 버튼 (향후)
- 저장 버튼 (향후)

**메인 영역** (스크롤 가능):
- 메뉴 대표 이미지
- 메뉴명 + 한 줄 소개
- 조리시간 / 난이도 / 칼로리
- [재료 정보] 섹션
- [조리 방법] 섹션
- [영양 정보] 섹션
- [팁 & 주의사항] 섹션

**하단** (고정):
- "다른 메뉴 보기" 버튼

## 기술 외 고려사항

### 1. 콘텐츠 (메뉴 데이터)

**필요한 콘텐츠**:
- 최소 50개 메뉴 레시피
- 각 메뉴별 이미지 (고품질 음식 사진)
- 상세한 조리법
- 정확한 영양 정보
- 재료별 가격 정보 (정기 업데이트 필요)

**콘텐츠 제작 방식**:
- 직접 요리 및 촬영
- 또는 레시피 전문가/영양사와 협업
- 또는 기존 레시피 DB 활용 (라이센스 확인)

### 2. 사용자 인터페이스 디자인

**디자인 방향**:
- 친근하고 따뜻한 느낌
- 음식 사진이 돋보이도록
- 깔끔하고 현대적인 레이아웃
- 가독성 높은 타이포그래피

**색상 팔레트**:
- 주색: 식욕을 돋우는 따뜻한 색 (오렌지, 그린 등)
- 보조색: 중성적이고 깔끔한 색

**아이콘 및 이미지**:
- 통일감 있는 아이콘 세트
- 고품질 음식 사진

### 3. 사용성 테스트

**MVP 출시 전 테스트**:
- 10-20명의 타겟 사용자 테스트
- 질문이 명확한가?
- 추천이 만족스러운가?
- 실제로 만들어보았는가?

**개선 포인트 수집**:
- 헷갈리는 질문 문구
- 추천 정확도
- 빠진 기능
- 불편한 UI/UX

## 성능 요구사항 (비기술적 관점)

### 1. 속도

**사용자 기대치**:
- 질문 화면 전환: 즉시 (0.5초 이내)
- 추천 결과 생성: 3초 이내
- 메뉴 상세 화면 로딩: 1초 이내

**목표**: 사용자가 "느리다"고 느끼지 않도록

### 2. 정확도

**추천 정확도 목표**:
- 추천받은 메뉴 중 최소 1개는 "만들어보고 싶다" 평가
- 기피 식재료 필터링 정확도 100%
- 예산 범위 초과 메뉴 0개

### 3. 안정성

**목표**:
- 서비스 중단 없이 원활한 사용
- 에러 발생 시 친절한 안내 메시지
- 사용자 입력 데이터 손실 방지

## MVP 핵심 기능 우선순위

1. **최우선** (없으면 서비스 불가):
   - 질문 응답 시스템
   - 식단 추천 (3개)
   - 메뉴 상세 정보

2. **높은 우선순위** (사용자 경험에 중요):
   - 다시 추천받기
   - 이전 질문으로 돌아가기
   - 프로그레스 표시

3. **중간 우선순위** (있으면 좋음):
   - 도움말 팝업
   - 추천 이유 표시
   - 재료 대체 제안

4. **낮은 우선순위** (MVP 이후):
   - 애니메이션
